# Adapters

Each adapter implements the `probe() → extract() → copy()` contract. The
adapter set is a **design target** for the phase-4 lifeboat pipeline (adapters →
Pass A → Pass B → Pass C); none of the dedicated adapter modules below have
shipped yet. Where the lifeboat's needs are already served by a shipped library
module, the Status column names it; the canonical module inventory is
[`scripts/abcd/README.md`](../../../../scripts/abcd/README.md).

**Convention:** semantic adapter name + per-vendor backend (see [`04-universal-patterns.md § 7`](04-universal-patterns.md#7-vendor-agnostic-adapters-with-environment-branching)). Consumers (agents) read from the semantic adapter; backends are implementation detail.

**Vendor-memory read-only invariant.** `memory.py` and its backends treat the vendor's memory directory (under Claude Code: `~/.claude/projects/<encoded-cwd>/memory/`) as a **read-only harvest source**. abcd reads it, distils it, and writes curated pages to its own `.abcd/memory/` — it **never writes to, edits, or deletes anything under `~/.claude/`**. That directory is the harness's domain and the user's. abcd keeps `~/.claude/` minimal: only the abcd plugin install lives there; all abcd-authored material routes to the scope-appropriate `.abcd/` (per [`03-configuration.md` § The two `.abcd/` scopes](03-configuration.md#the-two-abcd-scopes)). The `claude.py` backend's memory-format knowledge — it follows the `MEMORY.md` index to the per-fact files and reads their frontmatter — exists purely to *read* structure, not to round-trip it.

**Pattern note:** the dispatcher pattern in [`04-universal-patterns.md § 7`](04-universal-patterns.md#7-vendor-agnostic-adapters-with-environment-branching) (semantic name + per-vendor backends) applies only when **multiple vendor backends are anticipated**. Adapters with a single expected backend (`flow_next.py`, `claude_md.py`, `adr.py`, `git_log.py`, `assets.py`, `user_docs.py`, `work_dir.py`, `rp_state_backend.py`) are designed as direct implementations. The dispatcher form is reached for explicitly when a second vendor lands (e.g., itd-22 OpenCode will likely reshape `rp_state_backend.py` into a dispatcher).
