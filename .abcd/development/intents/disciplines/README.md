# Disciplines

Discipline-kind intents — cross-cutting rules with no user moment of their own.
They never get a native spec; instead they impose acceptance gates that every
*other* spec inherits and is checked against. There is no `status` frontmatter
field: presence in this directory IS the active state, and supersession moves the
file to `superseded/`.

See the [intents index](../README.md) for the corpus listing and the
[intent surface contract](../../brief/04-surfaces/05-intent.md) for the discipline
format.
