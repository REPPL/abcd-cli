# plans/

Dated design and implementation plans (`YYYY-MM-DD-topic.md`) — the "here's how we
will build X" artefacts. Date-prefixed because they are chronological, unlike ADRs
(stable sequential `NNNN` handles). A decision extracted from a plan graduates to an
ADR in [`../decisions/adrs/`](../decisions/adrs).

The Go-rebuild plan is promoted here during the Phase 0.5 content reconciliation.
